Assumptions — Steel emission-intensity (decoupling) model

Task IDNOX-003.1
Feature slugsteel-intensity-model
Created2026-06-13

Purpose

Explicit assumptions behind the intensity-decomposition method and the full-series re-run. The first is the identifying assumption of the whole approach; if it fails, the method honestly cannot separate intensity from activity, and that is a reportable outcome.

Assumptions

A1Timescale separation (identifying assumption)Load-bearing
Assumption
The emission-intensity decline (retrofit/policy decoupling, Li 2024) is slow and smooth relative to the activity variation we want to recover; i.e. the secular trend s(t) and the activity residual live on separable timescales.
Why it matters
This is what makes signal − s(t) ≈ activity identifiable. If intensity and activity share timescales, no smoothing separates them — double-differencing already demonstrated this failure mode on the weekly series.
If false
The df-sensitivity sweep (REQ-103) will show no stable plateau and the residual correlation will be smoothing-dependent → report as inseparable / null rather than forcing a number.
A2Smoothness can be chosen without the benchmarkMethod
Assumption
A defensible trend smoothness can be selected by cross-validation on the NO₂ series alone (prediction error), independent of how well the residual correlates with the CREA benchmark.
Why it matters
Selecting df against the benchmark would be circular / p-hacking (NFR-102). CV on the series itself keeps the estimator honest.
If false
If CV is unstable on the short series, fall back to a recorded fixed low df and lean on the sensitivity sweep for honesty; never tune df to the benchmark.
A3Retrofit decline is approximately continuous, not a stepMethod
Assumption
Over 2019→present the intensity decline is gradual enough that a smooth estimator captures it; discrete large plant upgrades (step changes) are second-order.
Why it matters
A single smooth trend mis-fits a sharp step (EDGE-103), biasing nearby residuals.
If false
Residual diagnostics flag the misfit; change-point / segmented trend modelling is noted as future work, out of this scope.
A4Methodology can be developed on synthetic + partial dataProcess
Assumption
The decomposition + CV + sweep + reporting can be implemented and tested on synthetic planted fixtures and the current partial cube; the complete-fetch re-run (REQ-120/121) is the final acceptance gate, not a prerequisite to start.
Why it matters
Lets the work proceed in parallel with the still-running TROPOMI fetch.
If false
If the fetch completes first, run REQ-120/121 directly; if not, deliver the harness + partial-labelled run and record the full-series number as a follow-up (AC-106).
A5Existing libraries sufficeLow risk
Assumption
scipy/statsmodels (already present) provide a df-controllable smoother (spline / LOWESS / GAM) and the building blocks for blocked CV, so no new dependency is required.
Why it matters
Keeps the increment dependency-free (NFR-106).
If false
Propose a single, well-maintained dependency at implementation for explicit developer approval (open question Q2).
A6Relative-index discipline still holdsInherited
Assumption
All NOX-003 constraints carry over unchanged: relative index only (no absolute tonnage), attributable-fraction cap recorded, TROPOMI low-bias caveat, null is acceptable, flux divergence deferred to NOX-005.
Why it matters
This task changes the detrending, not the interpretation discipline.
If false
n/a — these are locked by the prior approval.
