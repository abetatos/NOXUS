Acceptance Tests — GEMS NO₂ acquisition

Featuregems-acquisition
Task IDNOX-007
Created2026-06-14

Purpose

Observable behaviour required for acceptance. Network/auth are mocked with a synthetic granule; the real-data checks (AT0 measurement spike, AT5b co-location/diurnal) are developer-run on the server.

Test strategy

  • A fake HTTP layer returns a granule listing + a small synthetic NetCDF granule (a region covering the AOI, several scan slots per day) → deterministic unit tests, no network/key.
  • Clip/QA/coverage/version/resumability/volume-discipline asserted on the synthetic granule.
  • Verification: synthetic store for clear-scan selection + render + diurnal panel; real-data visual check is manual.
  • AT0 (the T1 size + go/no-go measurement) is recorded, not CI-asserted.

Test command

uv run pytest
uv run ruff check .

Manual acceptance checklist

  • T1 measurement + go/no-go recorded (AT0).
  • All requirements have a test (auto or manual).
  • Only AOI window kept; full granules deleted after clip.
  • No key logged/committed.
  • Real slice verified (NO₂ over sub-clusters + diurnal panel) (AT5b).
  • Unresolved behaviour captured in open-questions.html.

Notes for Claude Code

Rules

  1. Tests map back to requirements; auth/network mocked with a synthetic granule.
  2. Mark AT0 and AT5b clearly as manual (needs the key + server, non-deterministic).
  3. Do not invent behaviour beyond the approved spec.
