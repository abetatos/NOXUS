Summary

Every result so far (NOX-003/003.1/006, the deep exploration, and the 2026-06-14 autocorrelation probe) was computed at one fixed spatial scale: the default 0.25° AOI buffer and the native ~0.05° grid. The honest null we reported could therefore be an artefact of that single choice. Parubets & Naito (2025) showed exactly this risk for NO₂↔GDP in Japan — the relationship is significant at 0.25° but loses significance / flips sign at 0.1° — and warn that aggregation scale must be treated as a tuned hyper-parameter, not maximised. A competing intuition pulls the other way for our case: steel is only ~30–43% of Tangshan NO₂ (Wen 2024), so a tighter AOI could improve source isolation by excluding the Beijing-Tianjin-Hebei background.

This feature sweeps two scale axes over the existing cube and recomputes the NO₂↔BF-rate correlation at each scale with an autocorrelation-robust, multiplicity-corrected significance test, so the conclusion graduates from "no signal at one scale" to "no robust signal at any reasonable scale" — or surfaces the scale where a signal appears.

  • Axis 1 — AOI extent: default 0.25° buffer vs 0.10° tight buffer (a strict spatial clip of the cube; both constants already exist in noxus/config/region.py).
  • Axis 2 — grid resolution: native vs block-averaged to ~0.1° and ~0.25° — aggregation only, never interpolation (the 2026-06-13 decision rejected interpolating to a finer grid; coarsening by averaging is the permitted direction and is precisely the Parubets experiment).

Feasibility (verified 2026-06-14): the cube is 30×51 cells spanning the wide AOI (lon 117.74–119.32, lat 38.71–40.44); the tight AOI is a strict subset, and coarsening to 0.1°/0.25° is a block-mean of native cells — both are pure re-aggregations of the committed cube, no re-fetch.

Honesty (carried over): coarsening adds no information — it only changes how much background is blended in; an honest null across all scales is a valid, designed-for outcome (Morris & Zhang 2019). The robust-significance machinery (effective-N, block bootstrap, block permutation, FDR) is the load-bearing new capability and is reusable by the wider pipeline.

Functional requirements

EARS format. IDs are stable references for tasks and tests.

Scale construction (extent × resolution)

REQ-001When given the analysis-ready cube and an AOI Region, the system shall clip the cube to the cells whose centres fall within that region, returning a strict spatial subset without interpolation or resampling, so the 0.25°→0.10° tight-AOI comparison is a pure subset (REQ uses tangshan_aoi(0.25) vs tangshan_aoi(0.10)).
REQ-002When given a target resolution coarser than native, the system shall block-average the cube (mean over non-overlapping cell blocks sized to the target degree spacing), propagating coverage as the mean of contributing valid cells and emitting a block as NaN when it has no valid cell — aggregation only, never interpolation (consistent with the native-resolution decision, 2026-06-13).
REQ-003When coarsening, the system shall record the realised cell spacing and grid shape per target (native, ~0.1°, ~0.25°) so the actual resolution used is auditable and not assumed (EDGE-001).

Per-scale signal re-derivation

REQ-010When a scale (extent × resolution) is selected, the system shall re-derive the NO₂ activity signal from the clipped/coarsened cube using the existing footprint/AOI-mean + background pathway and the existing deseasonalize variants (level / intensity-model residual / yoy / stl), with no change to those algorithms (additive reuse).
REQ-011When a coarsened scale leaves too few cells for the footprint/background contrast (e.g. 0.25° over the tight AOI), the system shall fall back to the AOI spatial mean and label the signal accordingly, rather than fabricating a footprint from one cell (EDGE-002).

Autocorrelation-robust significance

REQ-020When testing a correlation for significance, the system shall compute an effective sample size that accounts for serial dependence — at minimum the first-order (Bayley–Hammersley) estimate and a full-order/Newey-West-style estimate — and report a p-value on the effective degrees of freedom alongside the naïve Pearson p (so the gap is explicit).
REQ-021When quantifying uncertainty in r, the system shall provide a moving-block bootstrap confidence interval (block length scaled to the series length, paired resampling that preserves within-block memory) with a configurable number of draws and a fixed seed.
REQ-022When testing the null "no cross-correlation", the system shall provide a block-permutation p-value (reorder contiguous blocks of one series against the other, preserving each series' own autocorrelation while destroying the cross-correlation), with a fixed seed.
REQ-023The robust-significance functions shall be added to noxus/validation/ as reusable, tested code (not analysis-script throwaways) and shall be additive: the existing correlate, lead_lag and verify_sign contracts are unchanged.

Multiple-testing correction

REQ-030When many correlations are tested across the scale × variant × lag grid, the system shall apply a Benjamini–Hochberg FDR correction over the family of p-values and report the adjusted significance, so an isolated "hit" among dozens of tests is not over-claimed.

Sweep, comparison & reporting

REQ-040When the sweep runs, the system shall produce a tidy result table with one row per (extent, resolution, variant, lag-choice) carrying: realised cell spacing, n, r, naïve p, effective-N p (1st & full order), block-bootstrap CI, block-permutation p, FDR-adjusted significance, and a robust verdict.
REQ-041When reporting, the system shall present the result as a scale grid (heatmap/figure of r and robust significance over extent × resolution) and shall state explicitly whether the data reproduce the Parubets pattern (significant coarse → lost/flipped fine) and whether the tight AOI improves coupling (source-isolation hypothesis).
REQ-042When the outcome is a null, the system shall report it as a valid finding ("no robust coupling at any swept scale"), with CIs, not as a failure — and shall not cherry-pick the most favourable scale without the multiplicity caveat (Morris & Zhang 2019).

CLI / orchestration

REQ-050When the user runs the sweep, the system shall expose it reproducibly (a noxus subcommand and/or an analysis/ driver) from a recorded config (resolutions, buffers, bootstrap draws, FDR α, seed); existing subcommands and defaults are unchanged.

Non-functional requirements

NFR-001Reproducibility. The whole sweep reproduces from the committed cube + recorded config; no live fetch at analysis time; bootstrap/permutation use fixed seeds.
NFR-002No interpolation. Both scale operations are pure re-aggregation of observed cells (clip = subset; coarsen = block-mean); cloud-gap cells stay NaN — the cloud/native-resolution decisions hold.
NFR-003Anti-p-hacking. The deseason/intensity smoothness is still selected on the NO₂ series alone (NOX-003.1 NFR-102); the scale sweep never selects the scale that maximises the benchmark correlation as if it were the headline result — every scale is reported with the FDR caveat.
NFR-004Additive. Existing validation/gridding contracts and CLI behaviour are unchanged; new code is opt-in.
NFR-005No-publish. NO₂-derived per-scale series/cubes are gitignored; small stats tables + figures may be committed.
NFR-006Stack conformance. Python ≥3.12; passes uv run pytest + uv run ruff check .; no new heavy dependency (SciPy/NumPy already present; FDR/Newey-West implemented locally or via existing statsmodels).
NFR-007Honest reporting. Coarsening adds no information; the report states this and frames any apparent gain as reduced background dilution, not new signal.

Edge cases

EDGE-001Target resolution not an integer multiple of native spacing → block edges don't align; the system rounds the block factor, records the realised spacing (≠ the nominal target) and reports it, never silently mislabels (REQ-003).
EDGE-002Coarse resolution × tight AOI leaves only a handful of cells → footprint/background contrast is ill-posed; fall back to AOI mean and label it (REQ-011); if <1 valid cell remains, that scale is dropped with a recorded reason, not crashed.
EDGE-003Block-permutation with very few blocks (short series ÷ large block) → too few distinct permutations; the system caps the block length and reports the effective permutation count so the p-value's resolution is honest.
EDGE-004Effective-N estimate degenerate (autocorrelation ≥ 1 within numerical error, or n_eff < 3) → clamp to a floor and report the clamp, returning a null-leaning p rather than a spurious significant one.
EDGE-005Partial cube (fetch incomplete) → the sweep still runs and labels the result partial; the full-fetch cube (NOX-003.1 T10, already rebuilt to 389 wks) is the arbiter.

Error states

ERR-001If the analysis-ready cube is missing, the sweep fails with an actionable message naming noxus grid.
ERR-002If a requested target resolution is finer than native, the system refuses with a message pointing at the native-resolution decision (no upsampling/interpolation), rather than fabricating cells (NFR-002).
ERR-003If an AOI clip yields no overlapping cells (region outside the cube), the system fails with an actionable message naming the cube extent and the requested bbox.

Acceptance criteria

AC-001Clipping the cube to tangshan_aoi(0.10) yields a strict cell subset of the 0.25° cube (fewer y/x cells, identical values on shared cells, no new/interpolated cells) (REQ-001).
AC-002On a synthetic cube with a known field, block-averaging to a 2×/5× coarser grid returns the exact block means, propagates coverage, and yields NaN for all-NaN blocks; realised spacing is recorded (REQ-002/003, EDGE-001).
AC-003On a synthetic autocorrelated null (two independent AR(1) series), the naïve Pearson p over-rejects at the nominal α while the effective-N, block-bootstrap and block-permutation tests do not (false-positive rate ≈ α) (REQ-020/021/022).
AC-004On a synthetic series pair with a planted lagged correlation, the robust tests recover it (CI excludes 0, permutation p < α) — i.e. they are not merely conservative (REQ-021/022).
AC-005Benjamini–Hochberg FDR on a known p-value vector matches the reference adjusted values; a family of all-null p's yields ~no discoveries (REQ-030).
AC-006The sweep emits the tidy table (one row per extent×resolution×variant×lag) with every column of REQ-040 and a scale-grid figure; re-running with the same seed/config reproduces it identically (REQ-040/041, NFR-001).
AC-007The report states the Parubets verdict (does coarse→fine behave as in Japan?) and the source-isolation verdict (does tight AOI help?), and frames a null as a valid finding with the multiplicity caveat (REQ-041/042).
AC-008The existing correlate/lead_lag/verify_sign behaviour and the existing CLI are unchanged (NFR-004); uv run pytest and uv run ruff check . pass (NFR-006).
