Technical summary

Two small reusable additions plus a thin orchestration layer. (1) Scale construction in noxus/data/gridding.py: clip_cube_to_region (strict cell subset for the AOI-extent axis) and coarsen_cube (block-mean to a target degree spacing for the resolution axis) — both pure re-aggregations of the committed cube, never interpolation. (2) Autocorrelation-robust significance in a new noxus/validation/robust.py: effective-N (first-order Bayley–Hammersley + a full-order Newey-West-style estimate), moving-block bootstrap CI of r, block-permutation null p, and Benjamini–Hochberg FDR — promoting the 2026-06-14 probe (analysis/autocorr_significance.py) into tested code, additive to leadlag.py. (3) A sweep driver (noxus/validation/scale.py + an analysis/ figure script) that, for each (extent × resolution) scale, re-derives the signal through the existing footprint/deseason pathway, aligns to the BF rate, and tabulates r + robust significance, then emits the scale-grid figure and a findings file.

Why this shape: the robust-significance functions have value beyond this task (they fix the i.i.d. assumption in every correlation the project reports), so they live in noxus/validation/ as tested code, not in analysis/. The scale operations belong next to the gridding they re-aggregate. The orchestration is deliberately thin and reuses NOX-003/003.1 unchanged.

Analogy caveat (load-bearing): our predictor is a single AOI-mean/footprint time series, so "resolution" here means how coarsely we average within the AOI before the footprint/background contrast — an analogue of Parubets' cross-prefecture panel, not an identity. This is recorded as assumption A1 and stated in the report.

Test design

  • Clip (AT1): a synthetic cube → clip_cube_to_region with the tight bbox returns a strict subset (fewer cells, equal values on shared cells, no interpolation).
  • Coarsen (AT2): a synthetic cube with a known field → block-mean to 2×/5× returns exact block means, propagates coverage, NaN for all-NaN blocks; realised spacing recorded (EDGE-001).
  • Robust null (AT3): two independent AR(1) series with strong autocorrelation → naïve Pearson p over-rejects across many seeds while n_eff/bootstrap/permutation hold the false-positive rate near α.
  • Robust power (AT4): a planted lagged-correlation pair → CI excludes 0 and permutation p < α (the tests are not merely conservative).
  • FDR (AT5): bh_fdr vs a hand-computed reference vector; all-null family → ~no discoveries.
  • Sweep (AT6): a small synthetic cube + benchmark → tidy table with all REQ-040 columns + a figure; identical re-run under the same seed (determinism).
  • Error states (AT-ERR-1): missing cube → actionable; finer-than-native target → refusal pointing at the decision; empty clip → actionable.

All statistics tested on synthetic fixtures (deterministic, seeded, no network). Bootstrap/permutation draws are reduced in tests for speed while asserting the rate behaviour.

Risks and trade-offs

  • Analogy is imperfect (stated). Parubets ran a cross-prefecture panel; we run a single AOI-mean time series, so "resolution" = within-AOI averaging coarseness. Mitigation: state it (A1); interpret a coarse↔fine difference as background-dilution, not a literal replication.
  • Coarse × tight AOI runs out of cells. 0.25° over the tight AOI is a few cells → footprint ill-posed. Mitigation: AOI-mean fallback + label (REQ-011); drop scales with <1 valid cell (EDGE-002).
  • Few independent samples ⇒ wide CIs. The robust tests will (correctly) widen CIs and may make even the level's negative r non-significant. That is the honest point, not a bug.
  • Permutation resolution on short monthly series. Few blocks → coarse p. Mitigation: cap block length, report effective permutation count (EDGE-003).
  • Scale-shopping temptation. Reporting the best scale without FDR would be p-hacking. Mitigation: FDR across the whole family; every scale reported (NFR-003).
  • Rejected: upsampling/interpolating to a finer grid to "gain resolution" — rejected by the 2026-06-13 native-resolution decision and refused at runtime (ERR-002).
