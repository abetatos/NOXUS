Acceptance Tests — Spatial-scale sensitivity of the NO₂↔steel signal

Task IDNOX-008
Feature slugspatial-scale-sensitivity
Statusspec_draft

Overview

Statistics verified on synthetic fixtures — deterministic, seeded, no network. Scale ops run on small synthetic cubes; the robust tests run on AR(1) fixtures with reduced bootstrap/permutation draws for speed while asserting the rate behaviour. The real-data verdict (AT7) is run once on the committed full-series cube.

Acceptance tests

AT1Clip = strict cell subset, no interpolation
Covers
REQ-001 · AC-001
Given
A synthetic cube on a regular grid and a tight Region strictly inside it.
Then
The clipped cube has fewer y/x cells, every retained cell's value equals the original (no new/averaged cells), and no coordinate falls outside the region.
AT2Coarsen = exact block-mean + coverage + realised spacing
Covers
REQ-002 · REQ-003 · AC-002 · EDGE-001
Given
A synthetic cube with a known field (and some NaN cells) coarsened by a 2× and a 5× block factor.
Then
Each coarse cell equals the mean of its native block over valid cells; an all-NaN block is NaN; coverage is the block mean; the realised cell spacing is recorded in .attrs (and may differ from the nominal target).
AT3Autocorrelated null is not over-rejected
Covers
REQ-020 · REQ-021 · REQ-022 · AC-003
Given
Many seeded pairs of independent AR(1) series with strong autocorrelation.
Then
The naïve Pearson p rejects at well above the nominal α, while the effective-N (1st & full order), block-bootstrap (CI contains 0) and block-permutation tests hold the false-positive rate near α.
AT4Planted signal is recovered (not merely conservative)
Covers
REQ-021 · REQ-022 · AC-004
Given
A series pair with a planted lagged correlation above the noise.
Then
The bootstrap CI excludes 0 and the block-permutation p < α; the lead-lag peak lands at the planted lag.
AT5Benjamini–Hochberg FDR
Covers
REQ-030 · AC-005
Then
bh_fdr on a reference p-value vector returns the hand-computed adjusted values and rejection set; an all-null family yields ~no discoveries; a family with a strong signal yields it as a discovery.
AT6Sweep table + scale grid + determinism
Covers
REQ-010 · REQ-011 · REQ-040 · REQ-041 · AC-006
Given
A small synthetic cube + benchmark over {0.25°,0.10°} × {native,coarse}.
Then
The tidy table has one row per (extent,resolution,variant,lag) with every REQ-040 column (incl. realised spacing, n_eff×2, bootstrap CI, perm p, FDR-adjusted sig, verdict); the AOI-mean fallback fires + is labelled when cells are too few; the scale-grid figure is produced; re-running under the same seed/config reproduces the table identically.
AT7Real-data verdict: Parubets + source-isolation, honest null
Covers
REQ-041 · REQ-042 · AC-007
Given
The committed full-series cube + CREA BF rate.
Then
The findings state whether the Parubets pattern holds (coarse significant → fine lost/sign-flipped) and whether the tight AOI improves coupling; a null is reported as a valid finding with the FDR/multiplicity caveat; no single scale is presented as the headline without that caveat.
AT8Additive — existing contracts + tests/lint
Covers
NFR-004 · NFR-006 · AC-008
Then
correlate/lead_lag/verify_sign and the existing CLI behave exactly as before; uv run pytest and uv run ruff check . pass.
