Technical summary

Mostly re-use + one new module. The daily series is the existing NOX-002b gridding at freq="D" followed by the NOX-003 footprint sampling — both already frequency-agnostic (verified: a daily cube of 2,720 days, 1,984 valid after screening, builds today). The genuinely new piece is a Prophet wrapper in noxus/signal/ that fits trend + yearly + weekly seasonality on the gappy daily footprint NO₂ and returns the components + the observed-only residual, wired in as an additive deseason_method="prophet".

Why Prophet here and not just more Fourier: Prophet gives, in one model, (a) a flexible piecewise-linear trend that removes the secular emission-intensity decline (the developer's "background reduction"), (b) yearly seasonality (the heating cycle), and (c) weekly seasonality — the day-of-week term that is the whole point: it is the temporal handle on source separation (steel baseload vs the traffic weekly cycle), and it only exists at daily resolution. Determinism is kept by MAP estimation (uncertainty_samples=0).

Two consumers: (1) a diagnostic comparison (does the daily-Prophet residual beat the weekly intensity/harmonic residual against the BF rate?) and (2) a catalyst re-run (NOX-004) on the cleaner residual. Any lead/market claim requires a causal/rolling Prophet refit; the full-series fit is diagnostic only.

Test design

  • Daily compositing: synthetic overpasses (or a synthetic daily cube) → daily footprint with NaN cloud gaps; valid fraction + weekday counts emitted (AT1).
  • Prophet decomposition: planted trend + yearly + weekly + activity + noise daily series → recovers trend/weekly/yearly within tolerance, residual ≈ activity, residual on observed days only (AT2); refit reproduces identically (AT3).
  • Weekday diagnostic: planted weekly cycle → amplitude + variance-removed recovered; flat-weekly → ~zero amplitude (AT4).
  • Comparison + catalyst: the Prophet residual flows through the existing compare + catalyst contracts (AT5).
  • Causal guard: rolling-refit residual at t unchanged by appended future days (AT6).
  • Lazy import / fallback: package imports without prophet; ERR-002 actionable; harmonic fallback works (AT7).

Prophet tests run on small synthetic series (fast, deterministic MAP); skipped with a clear marker if prophet is unavailable in CI, with the harmonic-fallback test always running.

Risks and trade-offs

  • Marginal-gain risk (likely). If steel really is baseload, the weekly component is small and the gain over the harmonic residual is modest — that itself is the (honest) finding. Mitigation: report the variance-removed + the comparison; accept a null marginal gain.
  • Changepoint over-fitting. Prophet's piecewise trend can chase noise (same risk as the NOX-003.1 spline). Mitigation: constrain changepoint_prior_scale, report sensitivity.
  • Daily is noisier per point. One overpass/day vs a weekly composite → more measurement noise; partly offset by 5–7× more points and Prophet's smoothing. Report on observed days only.
  • Look-ahead via in-sample Prophet. The full-series fit uses all data; fine for diagnosis, invalid for a tradeable lead. Mitigation: causal rolling refit for any lead/market claim (REQ-032), guard test.
  • Determinism / dependency weight. Prophet (cmdstanpy) is heavy and stochastic by default. Mitigation: MAP (uncertainty_samples=0), lazy import, harmonic fallback kept.
  • SNR ceiling unchanged for levels. Prophet cannot beat the 40% mean-share ceiling for a level estimator; its value is via the weekly source-separation (variance share) + the event framing. Rejected: claiming Prophet "fixes" the correlation.

Rollback plan

Additive: a new module + a new deseason branch + a daily output; existing weekly paths and defaults unchanged. Rollback = revert the feature branch and delete the gitignored daily artifacts; if prophet proves problematic, disable the branch (harmonic remains the method of record).
