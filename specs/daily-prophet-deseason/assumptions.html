Assumptions — Daily NO₂ aggregation + Prophet decomposition

Task IDNOX-006
Feature slugdaily-prophet-deseason
Created2026-06-13

Purpose

Explicit assumptions behind the daily-Prophet path. A1 (the weekly cycle separates steel from traffic) is the load-bearing hypothesis; if it fails the gain is null — itself a reportable finding.

Assumptions

A1The day-of-week cycle separates steel from trafficLoad-bearing
Assumption
Steel is baseload (runs 24/7, ~flat across the week) while traffic/other NO₂ sources carry a real weekly cycle; so a Prophet weekly seasonal term captures mostly non-steel, and removing it raises steel's variance share in the residual.
Why it matters
This is the only temporal lever (short of flux divergence) that can beat the 40% mean-share ceiling.
If false
If the weekly cycle is weak or also carries steel structure, the gain over the harmonic residual is small/zero — reported honestly via the variance-removed + comparison (a valid null).
A2Daily coverage is weekday-unbiasedVerified
Assumption
Cloud cover does not systematically remove particular weekdays, so the weekly term is not aliased by sampling.
Why it matters
A weekday-biased sample would put a spurious weekly cycle into the data.
If false
The weekly component is flagged unreliable. Checked 2026-06-13: balanced (Mon 290 … Sun 270) — holds.
A3Prophet's trend removes the emission-intensity declineMethod
Assumption
The piecewise-linear Prophet trend captures the secular retrofit-driven NO₂ decline as well as (or better than) the NOX-003.1 spline.
Why it matters
The "background reduction" must be removed so the residual is activity, not trend.
If false
Tune changepoint prior; compare to the intensity spline; report sensitivity (EDGE-004).
A4Daily noise is tolerableData
Assumption
One overpass/day is noisier than a weekly composite, but 5–7× more points + Prophet's smoothing make the daily decomposition usable.
Why it matters
If per-day noise dominates, the weekly term is unidentifiable.
If false
Fall back to a 2–3-day rolling daily series, or to the weekly harmonic method.
A5In-sample fit for diagnosis, causal refit for leadsDiscipline
Assumption
The full-series Prophet fit is acceptable for the comparison/diagnostic; any lead/market claim uses the causal rolling refit (REQ-032).
Why it matters
The in-sample fit uses future data — invalid for a tradeable lead.
If false
n/a — enforced by design + the guard test.
A6SNR ceiling standsInherited
Assumption
Prophet does not change the physics: a level estimator is still capped near √(variance share); the win, if any, is raising that share via source separation, consumed through the event/catalyst framing.
Why it matters
Sets honest expectations — Prophet sharpens, it does not manufacture signal.
If false
n/a.
