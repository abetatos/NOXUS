Purpose

Observable behaviour required for acceptance. Network/auth are mocked; the one real-data check (AT5b) is the developer's clear-day verification.

Test strategy

  • A fake openEO connection returns a synthetic AOI overpass (xarray) → deterministic unit tests, no network/credentials.
  • QA/coverage/version/resumability asserted on synthetic windows.
  • Verification: synthetic store for selection + render; real-data visual check is manual.

Manual acceptance checklist

  • All requirements have a test (auto or manual).
  • Only AOI window transferred; no full granules.
  • No secrets logged/committed.
  • Real clear-day verification reviewed (AT5b).
  • Unresolved behaviour captured in open-questions.html.

Notes for Claude Code
