Acceptance Tests — Steel-sector NO₂ activity signal + validation

Featuresteel-activity-attribution
Task IDNOX-003
Created2026-06-13

Purpose

Observable behaviour required for acceptance. Fully offline — deterministic synthetic cube, facility CSV, synthetic ERA5, and planted-signal / planted-null fixtures. No network (ERA5 mocked).

Test strategy

  • Synthetic weekly cube with known cell coordinates/values + coverage; small facility CSV (operating cell, retired row, co-located duplicate).
  • Assert footprint cell selection, background correction, meteo regress-out, deseasonalisation, relative index, and validation against planted signal / planted null.
  • Statistical correctness via planted fixtures only — never asserted on live data.

Manual acceptance checklist

  • All requirements have a test.
  • Planted-null test proves honest null reporting.
  • No interpolation across cloud gaps.
  • Relative index only — no absolute tonnage.
  • Sign verified empirically; CIs reported.
  • ERA5 + NO₂-derived outputs gitignored; credentials by var name only.

Notes for Claude Code

Rules

  1. Tests map back to requirements; fully offline (synthetic fixtures, ERA5 mocked).
  2. The planted-signal / planted-null pair (AT6) is the headline determinism + honesty guarantee.
  3. Do not invent behaviour beyond the approved spec; do not assert statistics on live data.
