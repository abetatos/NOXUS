Q1Footprint radius / definition and background-ring geometryNon-blocking
Question
What radius (km) defines the facility footprint, and what inner/outer radii (or AOI-minus-footprint) define the regional background ring? At ~5 km native resolution a facility spans ~1 cell, so the footprint is a small neighbourhood.
Why it matters
Too small → noisy/empty footprint; too large → background-dominated, diluting the signal (Parubets 2025 scale warning; Kim 2023 background separation).
Blocking
No
Default assumption if not answered
Footprint = cells within ~10 km of an operating facility; background = annular ring ~20–50 km around the cluster, inside the AOI, excluding footprint cells. Tune both against the real coverage/signal and report sensitivity.
Affected files / sections
footprint_mask, background_ring; SignalConfig; REQ-002, REQ-003
Decision
Pending

No

Q3ERA5 variables / levels and regression formNon-blocking
Question
Source — RESOLVED (2026-06-13): ERA5 is ingested only from the Copernicus CDS (reanalysis-era5-single-levels), subsetting the AOI/era/overpass-hour server-side (only ~MB transferred; verified live, 42 KB for a 3-day subset). The anonymous ARCO-ERA5 Zarr store was considered and rejected: its chunk-1 layout downloads a full global field per timestep (~30–760 GB for the 2019→present AOI series), the wrong access pattern here. CDS needs a free account credential (CDSAPI_KEY in .env, by reference only) + one-time licence acceptance. Still open: which ERA5 fields (10 m u/v wind, boundary-layer height, 2 m temperature, others?) at which level, aggregated to which time window (overpass ~13:30 local vs daily mean), and what regression form — linear residualisation vs LOESS high-frequency filtering (Mao 2025)?
Why it matters
This is the dominant weekly-ventilation confounder; the wrong variable set or form leaves weather in the signal or over-removes activity.
Blocking
No
Default assumption if not answered
u10, v10, blh at the overpass window; linear residualisation as default with LOESS available; record the form. Refine in T5.
Affected files / sections
era5_footprint_series, regress_out_meteo; SignalConfig; REQ-010, REQ-011
Decision
Source resolved (Copernicus CDS; ARCO rejected) Variables/form pending
Q4Deseasonalisation method of recordNon-blocking
Question
RESOLVED (2026-06-13, real sensitivity run): default changed to yoy (single year-over-year change). The 2026-06-13 battery showed yoy-double-diff erased the signal on this weekly series (r≈0 — over-aggressive, as Kim 2023 warned), while yoy/stl preserved the faint positive activity signal after the secular trend is removed. All four methods (yoy/stl/yoy-double-diff/none) remain selectable for replicability; monthly > weekly. The heating-season term is applied as a structural covariate before deseasonalisation. See docs/preliminary-results.html.
Why it matters
The choice is decisive for whether a clean sector correlation emerges (Kim 2023). Double-differencing erased it here.
Blocking
No
Default assumption if not answered
n/a — resolved to yoy.
Affected files / sections
deseasonalize; SignalConfig; REQ-020, REQ-021
Decision
Resolved — default yoy (double-diff demoted)
