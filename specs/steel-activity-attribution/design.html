Task IDNOX-003
Feature slugsteel-activity-attribution
Statusspec_draft

Technical summary

Four pipeline stages, each behind a small data contract so each is testable in isolation, ending in the revived validation engine. The chain realises the relative-index / cluster paradigm (Kim 2023), not flux divergence (Beirle; deferred to NOX-004).

  1. Attribution / footprint sampling (noxus/attribution/source.py): read the NOX-002b weekly cube + facility CSV; build a footprint mask around operating facilities; estimate a regional background from an annular ring; emit a per-period background-corrected footprint signal with propagated coverage.
  2. Meteorological normalisation (new noxus/data/era5.py ingest + residualisation in noxus/signal/): ingest ERA5 wind/PBL from the Copernicus CDS (server-side AOI subset) to a dated snapshot; aggregate to the footprint + frequency; regress it out of the footprint signal.
  3. Index construction (noxus/signal/index.py): deseasonalise (with an explicit heating-season term and curtailment exogenous control), normalise to a relative index, record the attributable-fraction cap.
  4. Validation (noxus/validation/): align to CREA, verify the sign, compute Pearson r + p, cross-correlation/lead-lag, emit results + a plain-language summary that reports the null and classifies against the r≈0.5–0.75 bar.

The validation stage revives the module reverted in NOX-001. The NOX-001 design specified a heavier econometric engine (STL/calendar deseasonalisation, ADF/KPSS stationarity gate, VAR/VECM, pseudo-real-time OOS + Diebold–Mariano). For NOX-003 v1 the headline tests are Pearson r + p and lead/lag as the task mandates; the heavier OOS/DM machinery from NOX-001 is reusable but is treated as an optional extension here so v1 first establishes whether any correlation exists at all (see Risks).

Test design

  • Footprint sampling: synthetic cube with known cell coordinates + a small facility CSV (incl. a retired row and a co-located duplicate) → assert the mask selects exactly the in-radius operating cells, dedup works, footprint value = manual mean (AT1).
  • Background: planted background field → corrected signal equals manual (footprint − ring); empty ring raises ERR-003 (AT2).
  • Meteo regress-out: signal = activity + β·wind + noise (synthetic ERA5) → residual recovers activity to tolerance; form recorded (AT3).
  • Deseasonalise / confounders: planted seasonal term removed; heating-season indicator + curtailment control present and recorded (AT4).
  • Index: output is normalised/unitless, cap recorded, no absolute tonnage (AT5).
  • Validation — planted signal & planted null: benchmark = f(index at lag k) + noise → recover sign, lag k, r in band; independent series → r≈0, p n.s., reports the null. This is the headline determinism guarantee (AT6).
  • Errors: missing cube/benchmark/ERA5; degenerate geometry; short sample (AT-ERR-1).
  • CLI: subcommands wired, config-driven, existing commands unchanged (AT-REG-1).

Statistical correctness is tested via planted-signal and planted-null synthetic fixtures, never by asserting on live data (which is non-deterministic). No network in tests (ERA5 mocked/synthetic).

Risks and trade-offs

High-risk-category note for the reviewer: this task is a large, cross-cutting feature that revives a reverted module and adds a new authenticated external data dependency (ERA5 via the Copernicus CDS; credential handled by reference only). The reviewer should apply the deep-review escalation ladder (reference/deep-review-policy.md): scrutinise the statistical claims (no p-hacking across the many configurable knobs), the relative-not-absolute discipline, the null-reporting path, and the credential handling.

  • Confounding residue. Meteorology, heating-season source-share shift (42.5%→29.1%, Wen 2024), and curtailments may not be fully removed; residual confounding can masquerade as — or mask — a real signal. Mitigation: explicit controls (REQ-021/022), report sensitivity to the covariate/knob set.
  • Researcher degrees of freedom. Footprint radius, background geometry, meteo form, deseason method and lag window are all tunable → risk of selecting a config that manufactures a positive r. Mitigation: fix defaults before looking at the correlation, record the full config in provenance, report the null honestly (NFR-003, Morris & Zhang 2019).
  • Sign instability. The NO₂↔activity sign is not fixed (Montgomery 2018); v1 must verify it empirically (REQ-041) and accept a negative/insignificant outcome.
  • Short / cloud-gapped sample. 2019→present weekly with coverage screening limits power and lag orders. Mitigation: cap lags, report CIs/power, ERR-004 on too-short overlap.
  • Low bias + source mixing. TROPOMI −23…−33% in polluted air (Kondragunta 2021) and steel only ~30–43% of the column (Wen 2024) → strictly relative index, cap recorded (REQ-030/031). Rejected: any absolute steel-tonnage claim.
  • Paradigm risk (designed-around). The relative-index v1 may itself produce a null; that is the explicit reason flux divergence is a separate later phase (NOX-004), gated on whether v1 finds any signal. Rejected here: flux divergence / raw-L2 oversampling (out of scope, deferred).
  • Validation depth trade-off. v1 leads with Pearson r + lead-lag per the task; the heavier NOX-001 OOS+Diebold–Mariano engine is reusable but deferred to keep v1 focused on "does a correlation exist at all". Risk: in-sample r alone over-claims → mitigated by the sign check, CIs, null reporting, and the r≈0.5–0.75 bar classification.
