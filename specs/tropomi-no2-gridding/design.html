Technical summary

A pure-local (no network) pipeline that reads the NOX-002a per-overpass NetCDFs via the manifest, stacks them into a time cube on a common grid (the openEO native ~0.05°×0.035° grid by default; resample only if a target grid is configured or grids differ), composites to a target frequency (weekly default) by the mean over valid cells, computes a coverage layer, applies the minimum-coverage threshold (mask, never interpolate), and writes a NetCDF cube + an interim AOI-mean parquet series under a gitignored data/derived/no2/.

Because the input is already gridded, there is no swath oversampling here; the work is xarray stacking + resample(time).mean() + coverage bookkeeping. Fine (~1 km) oversampling is a separate L2 track (NOX-003), not in scope.

Data model changes

New gitignored derived store data/derived/no2/ (NetCDF cube + parquet series). No database. Note: NetCDF is already gitignored globally (*.nc); the parquet needs the explicit data/derived/no2/ rule.

External dependencies and freshness

None new — pure local processing with existing numpy/pandas/xarray (+ h5netcdf for I/O). No network.

Security and permissions

None. Local files only; no secrets, no network, no protected files. NO₂-derived outputs gitignored so downloaded-data derivatives are not published.

Documentation targets

Project map (gridding implemented, new derived store), .gitignore, optionally a line in docs/data-access.md describing the cube/series. Reviewer confirms.

Test design

  • Synthetic store: write a handful of per-overpass NetCDFs (known values/dates) + manifest; assert the weekly cube equals the manual weekly means and the coverage layer is correct (AT1, AT2).
  • Threshold: a sparse week below the coverage threshold is masked; a dense week is kept; no interpolation introduced (AT2).
  • AOI series: equals the spatial mean over valid cells per period; carries predictor columns + naive label (AT3).
  • Errors: empty store; irreconcilable grid (AT-ERR-1).
  • CLI: grid wired, config-driven (AT-REG-1).

UI verification plan

None (no UI). Optionally reuse verify-no2 rendering on a composited week for eyeballing.

Risks and trade-offs

  • Native resolution only. Cube is ~5 km (TROPOMI native) — adequate for the cluster signal, not per-plant. Fine oversampling deferred (L2 path, NOX-003). Decision A (2026-06-13).
  • Interim AOI mean is not attribution. It is a naive spatial average to unblock end-to-end validation testing; must be clearly labelled to avoid being mistaken for the real predictor.
  • Version discontinuity. Carried through and surfaced in provenance; the validation stage must control for it (it does, via the v1→v2 awareness).
  • Rejected: interpolating cloud gaps (violates the cloud decision); swath oversampling here (input is already gridded; needs L2).

Rollback plan

Additive new module + CLI subcommand. Rollback = revert the branch and delete data/derived/no2/ (gitignored). Nothing downstream depends on it yet.
