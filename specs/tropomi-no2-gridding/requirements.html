Summary

Turn the per-overpass AOI NO₂ from NOX-002a into an analysis-ready cube: a time-stacked, temporally-composited, coverage-screened NO₂ grid over the Tangshan AOI, realising the cloud-handling decision (aggregation + minimum-coverage threshold, never interpolation; 2026-06-13).

Scope reality: the NOX-002a/openEO product is already on a regular grid (~0.05°×0.035°, ≈5×4 km, 30×51 cells over the default AOI). So this task is harmonisation + temporal compositing, not swath→grid oversampling. Beirle-style fine oversampling (sub-pixel resolution enhancement) needs raw L2 footprints and is deferred to the L2 path in NOX-003.

It also emits an interim AOI-mean weekly NO₂ series — a plain spatial mean over valid cells, explicitly pre-attribution — matching the validation predictor contract so the validation engine (NOX-001) can be smoke-tested end-to-end on real data before proper attribution exists. NO₂-derived outputs are gitignored (not published), per the developer's no-publish-downloaded-data decision.

Functional requirements

Cube assembly & compositing

REQ-001When gridding runs, the system shall load every overpass recorded in the NOX-002a manifest and stack them into a single time cube on a common grid; if overpass grids differ, it shall resample onto a configured target grid (default: the native openEO grid).
REQ-002When compositing, the system shall aggregate per-overpass NO₂ to a configured frequency (default weekly W; monthly optional) by the mean over valid (non-NaN) observations per cell.
REQ-003When compositing, the system shall compute and store a valid-coverage layer: per cell-and-period (fraction of contributing overpasses with a valid value) and a per-period AOI coverage summary.
REQ-004While compositing, the system shall apply a configurable minimum-coverage threshold (period-level and cell-level) and mask cells/periods below it to NA — never interpolating across cloud gaps (decision 2026-06-13).
REQ-005When the cube is built, the system shall write it to data/derived/no2/ as a NetCDF cube (time × y × x) of no2 + coverage with provenance (AOI, frequency, threshold, source manifest, processor versions present).

Interim AOI series

REQ-010When requested, the system shall emit an interim AOI-mean NO₂ series at the composite frequency — the spatial mean over valid cells per period, with the period valid-coverage — to data/derived/no2/no2_aoi_mean.parquet.
REQ-011The interim series shall match the validation predictor contract (columns date, no2_attributed, valid_coverage) but shall be clearly labelled as a naive spatial mean, not attribution (provenance/column note), so it is not mistaken for the NOX-003 product.

CLI

REQ-020When the user runs gridding via the noxus CLI, the system shall expose it as a documented subcommand driven by noxus/config/ (frequency, coverage threshold, target grid, input store, output dir).

Non-functional requirements

NFR-001Reproducibility. The cube is a deterministic function of the acquired store + config; provenance recorded.
NFR-002No-publish. NO₂-derived outputs (cube, AOI series) are gitignored — they are derived from downloaded data and must not be published.
NFR-003Cloud-decision conformance. Aggregation + minimum-coverage only; no gap interpolation in the production cube.
NFR-004Stack conformance. Python ≥3.12; numpy/pandas/xarray; passes uv run pytest and uv run ruff check ..

Edge cases

EDGE-001Periods with no valid overpasses (heavy cloud) → period masked, recorded as zero coverage, not dropped silently.
EDGE-002A single overpass in a period → composite = that overpass (coverage reflects 1 contributing day).
EDGE-003Overpass grids differ slightly (different AOI buffer / resolution) → resample to the target grid before stacking (REQ-001).
EDGE-004Mixed processor versions across the record (v1/v2) → carried through and surfaced in provenance (do not silently blend the discontinuity).
EDGE-005The store is partial (full fetch still running) → grid what exists; re-running after more data extends the cube.

Error states

ERR-001If the NOX-002a store/manifest is empty or missing, the system shall fail with a clear message ("run 'noxus fetch' first").
ERR-002If overpass grids cannot be reconciled onto the target grid, the system shall raise a descriptive error rather than silently mis-aligning.

Acceptance criteria

AC-001Given a synthetic per-overpass store, gridding produces a weekly cube (time × y × x) whose values equal the manual mean over each week's valid overpasses.
AC-002Periods/cells below the coverage threshold are NA; no interpolation is performed.
AC-003The interim AOI-mean series equals the spatial mean over valid cells per period and carries the predictor-contract columns + the "naive, not attribution" label.
AC-004Cube + series are written under data/derived/no2/ and are gitignored.
AC-005uv run pytest and uv run ruff check . pass.
