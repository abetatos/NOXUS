Purpose

Acceptance-level scenarios describing the observable behavior required for the feature to be accepted. The implementer turns these into automated tests; the reviewer verifies coverage.

Test strategy

  • Benchmark parsing tested against a committed fixture CSV reproducing the real hazards.
  • Statistical correctness tested with synthetic fixtures: a planted-lead dataset (must detect the lead) and a planted-null dataset (must report no lead). No assertions on live, non-deterministic data.
  • Error and CLI behavior tested directly.

Testing style configured for this project

pytest unit + acceptance tests; deterministic fixtures (fixed seeds); no network in tests
(fetch is mocked / fixture-backed).

Test command

uv run pytest
uv run ruff check .

Acceptance tests

AT1 Benchmark ingest produces a clean tidy parquet Draft
Related requirements
REQ-001, REQ-002, REQ-003, REQ-005, REQ-006
Scenario
Ingest from the fixture CSV.
Preconditions
tests/fixtures/crea_wind_sample.csv present.
Action
Run load_benchmark + emit_benchmark (or noxus ingest-benchmark --from-snapshot).
Expected result
Parquet has columns date,value,source,snapshot_date; Tangshan column selected by name; rows sorted ascending and de-duplicated on date; configured auxiliary series retained.
Negative expectations
No positional-index column selection; no metadata header rows leak into data.
Suggested test location
tests/data/test_benchmark.py
AT2 Preprocessing aligns, deseasonalizes, gates stationarity, residualizes Draft
Related requirements
REQ-010, REQ-011, REQ-012, REQ-013
Scenario
Synthetic NO₂ + benchmark with known seasonal term, trend, and a meteo covariate.
Preconditions
Coverage column present; some weeks below threshold.
Action
Run align_series, deseasonalize, stationarity_gate, residualize.
Expected result
Series aligned to weekly; sub-threshold weeks masked NA; seasonal term removed; ADF/KPSS reported and the model track chosen accordingly; meteo covariate's contribution removed from the NO₂ residual.
Negative expectations
Sub-threshold weeks are not aggregated from too few observations.
Suggested test location
tests/validation/test_preprocess.py
AT3 Planted lead is detected end-to-end Draft
Related requirements
REQ-020, REQ-021, REQ-022, REQ-023
Scenario
Synthetic data where benchmark = f(NO₂ at lag k) + noise.
Preconditions
Fixed seed.
Action
Run CCF, Granger, model fit, and OOS skill.
Expected result
CCF peak at lag k; Granger significant after correction; model coefficient at lag k significant; OOS AR+NO₂ RMSE < AR RMSE with a significant Diebold–Mariano statistic.
Negative expectations
No spurious peak at lag 0 when the true lead is k.
Suggested test location
tests/validation/test_oos.py

Manual acceptance checklist

  • All requirements have at least one acceptance test.
  • Main success path (ingest → validate → summary) covered.
  • Zero-as-missing edge covered.
  • Invalid input / error cases covered.
  • Null-reporting behavior covered.
  • Reconstruction isolation (non-goal) protected.
  • Tests feasible with current setup (synthetic fixtures, mocked fetch).
  • No acceptance test depends on hidden assumptions.
  • Unresolved behavior listed in open-questions.html.
