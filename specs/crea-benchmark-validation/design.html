Design — CREA benchmark ingestion + NO₂ activity-signal validation

Task ID NOX-001
Feature slug crea-benchmark-validation
Status spec_draft

Technical summary

Two pipeline stages, separated by a parquet contract so each is testable in isolation.

Benchmark side (noxus/data/benchmark.py): a small, pure-Python fetch → snapshot → parse → clean → tidy-parquet pipeline. The fetch writes a dated raw CSV; all downstream steps read the snapshot, never the network, so analysis is reproducible (NFR-001). The parser is hardened against the two known data hazards: multi-row headers and 0.00-as-missing placeholders. Column selection is by exact header name.

Validation side (noxus/validation/): a deterministic econometric pipeline built on statsmodels. It loads the NO₂ predictor (with coverage), aligns to a common frequency, deseasonalizes, residualizes against meteorology and curtailment controls, runs lead/lag diagnostics (CCF, Granger), fits the model track chosen by a stationarity gate (distributed-lag/VAR vs. VECM/ECM), and — the headline — runs a pseudo-real-time out-of-sample comparison of an AR baseline vs. AR+NO₂ with a Diebold–Mariano test. It emits a results artifact plus a plain-language summary that states the null when present.

Reconstruction (gap-filled NO₂) is a parallel, opt-in robustness input only; it never feeds the default path (decision 2026-06-13).

Security and permissions

None sensitive. Source is public and unauthenticated; no secrets, no protected files, no remote mutations. Network egress is limited to the CREA CSV (and, if approved, ERA5). No data is published externally.

Test design

  • Benchmark unit tests against a committed fixture CSV that reproduces the real hazards (multi-row header, trailing spaces, 0.00 placeholders, interleaved cadences): assert by-name selection, zeros→NA, sort/de-dup, parquet schema (AT1, AT-EDGE-1).
  • Preprocessing tests: synthetic series with known seasonality/trend → deseasonalization removes the seasonal term; stationarity gate routes correctly (AT2).
  • Stat tests: synthetic data with a planted lead (benchmark = f(NO₂ at lag k) + noise) → CCF recovers lag k; Granger significant; OOS AR+NO₂ beats AR. And a null dataset (independent series) → no lead, DM not significant: proves the pipeline reports the null (AT3, AT4).
  • Robustness test: reconstructed predictor path runs and is isolated from the default (AT5).
  • Error tests: unreachable URL, missing column, short sample, missing dependency (AT-ERR-1).
  • CLI smoke: subcommands invoke and honor config (AT-REG-1).

Statistical correctness is tested via planted-signal and planted-null synthetic fixtures, not by asserting on live data (which would be non-deterministic).

Risks and trade-offs

  • Endpoint instability. The CREA sheet could move or change shape. Mitigation: dated snapshots as source-of-record; by-name column selection; a clear ERR-002 on drift.
  • Short sample. ~2018→present limits lag orders and OOS folds; risk of underpowered tests. Mitigation: weekly frequency for more obs; lag caps; report power/CIs, not just point estimates.
  • Confounding residue. Meteorology and curtailments may not be fully removed; residual confounding could masquerade as (or mask) a lead. Mitigation: explicit controls, report sensitivity to covariate set.
  • Provenance. Upstream is commercial (WIND). Mitigation: document; rely on CREA's public republication; the project's "public/free" criterion is met at the access layer.
  • Rejected alternatives (see decisions/): HTML scraping of CREA reports (fragile; superseded by the CSV), commercial APIs (CEIC/Mysteel — paywalled, violates project constraint), reconstruction in the default path (injects model artifacts into the dependent variable — already rejected 2026-06-13).

Rollback plan

Both stages are additive new modules behind new CLI subcommands; no existing behavior changes. Rollback = revert the feature branch and remove the new derived artifacts. The committed snapshot and parquet can be deleted without affecting other modules. If statsmodels proves problematic, the dependency is removed and the validation subcommand disabled, leaving benchmark ingestion intact.
