Findings

None.

NBK-1A JSON-serialisation bug (numpy int64 keys in the Granger result) was found during end-to-end CLI verification and fixed; a regression test (test_results_artifact_is_json_serialisable) was added. Lesson: exercise the serialisation path, not only the in-memory result.
NBK-2Auxiliary benchmark columns are best-effort: absent ones are skipped (only the primary column is mandatory) so ingestion survives source column drift. This is a deliberate refinement of REQ-006's strict reading; documented here and in load_benchmark.
NBK-3Statistical tests use deterministic synthetic fixtures (planted-lead / planted-null), not live data, per the design. Real end-to-end validation awaits the upstream NO₂ attribution stage (open question Q1, default accepted).

Notes

UPDATE 2026-06-13 Validation reverted as premature. This review originally approved both benchmark ingestion and the validation engine. On the same day the developer (correctly) observed that validating made no sense with no predictor to validate — the NO₂ attribution stage does not exist yet, so the engine only ran on synthetic fixtures. The validation modules (preprocess/models/oos/report.py), the statsmodels dependency, and docs/validation-methods.md were reverted; leadlag.py returned to scaffold. Standing delivered scope: benchmark ingestion only (13 tests pass, lint clean). The validation design is retained in this spec for reuse and is deferred to a post-attribution task (NOX-003).

Original (pre-revert) note: scope per developer decision (Benchmark CREA + validation) with Q1–Q4 defaults accepted; the validation engine was complete and proven on synthetic fixtures.
